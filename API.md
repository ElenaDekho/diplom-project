# API Документация проекта

**Базовый URL:** `http://127.0.0.1:8000/api/`

Примечание: Все пути ниже указаны относительно базового URL. 
Например, путь /login/ означает полный адрес http://127.0.0.1:8000/api/login/.

## Аутентификация

Для доступа к защищенным эндпоинтам необходимо передавать токен в заголовке запроса:
Authorization: Token {ваш_токен}

### 1. Регистрация и Вход

#### Регистрация нового пользователя

**POST** `/register/`

Регистрирует нового пользователя и отправляет письмо с ссылкой для подтверждения email.

**Тело запроса:**

{
    "email": "user@example.com",
    "password": "123456",
    "first_name": "Иван",
    "last_name": "Иванов"
}

**Ответ (201 Created):**

{
    "message": "Пользователь создан"
}

#### Подтверждение Email

**GET** /confirm-email/{token}/

Активирует аккаунт после перехода по ссылке из письма.

**Ответ (200 OK):**

{
    "message": "Email подтверждён"
}

Токен действителен 24 часа. 
После подтверждения пользователь становится активным.

#### Вход (логин)

**POST** /login/

Возвращает токен авторизации.

**Тело запроса:**

{
    "email": "user@example.com",
    "password": "secure_password"
}

**Ответ (200 OK):**

{
    "token": "abc123xyz..."
}

#### Выход (Логаут)

**POST** /logout/

Удаляет токен текущей сессии. Требуется авторизация.

**Ответ (200 OK):**

{
    "message": "Вы вышли из системы"
}

### 2. Восстановление пароля

#### Запрос на сброс пароля

**POST** /password-reset/

Генерирует токен сброса пароля.

**Тело запроса:**

{
    "email": "user@example.com"
}

**Ответ (200 OK):**

{
    "reset_token": "uuid-token-here"
}

(В реальном приложении токен отправляется на email, здесь возвращается для демонстрации).

#### Подтверждение сброса пароля

**POST** /password-reset/confirm/

Устанавливает новый пароль по токену.

**Тело запроса:**

{
    "token": "uuid-token-here",
    "new_password": "new_secure_password"
}

**Ответ (200 OK):**

{
    "message": "Пароль изменён"
}

### 3. Товары и Магазины

#### Список товаров

**GET** /products/

Возвращает список всех доступных товаров (ProductInfo). Доступно без авторизации.

**Параметры фильтрации (query params):**

- shop={id} — фильтр по ID магазина.
- product__category={id} — фильтр по ID категории.
- search={text} — поиск по названию товара.

**Ответ (200 OK):**

[
    {
        "id": 66,
        "product_name": "Игрушка для кота",
        "shop_name": "ЗооМир",
        "price": 120,
        "price_rrc": 150,
        "quantity": 8,
        "parameters": [
            {
                "parameter": 7,
                "value": "Красный"
            },
            {
                "parameter": 12,
                "value": "Мышка шуршащая"
            }
        ]
    }
]

#### Детали товара

**GET** /products/{pk}/

Возвращает детальную информацию о конкретном товаре (ProductInfo).

**Ответ (200 OK):**

Аналогично элементу списка выше.

#### Магазины для конкретного товара

**GET** /products/{product_id}/shops/

Возвращает список магазинов, в которых продается товар с указанным product_id (ID модели Product, а не ProductInfo).

**Ответ (200 OK):**

[
    {
        "product_info_id": 66,
        "shop_name": "ЗооМир",
        "price": 120,
        "price_rrc": 150,
        "quantity": 10
    }
]

#### Управление статусом магазина (Поставщик)

**PATCH** /shops/{pk}/state/

Позволяет поставщику открыть или закрыть свой магазин для приема заказов.

Требования: Токен поставщика, магазин должен принадлежать пользователю.

**Тело запроса:**

{
    "state": true
}

(true — открыт, false — закрыт)

**Ответ (200 OK):**

{
    "message": "Магазин ЗооМир открыт"
}

### 4. Корзина и Контакты

#### Контакты пользователя

**GET** /contacts/ — Получить список контактов.

**POST** /contacts/ — Добавить новый контакт.

**DELETE** /contacts/{pk}/ — Удалить контакт.

**Тело POST запроса:**

{
    "city": "Москва",
    "street": "Тверская",
    "house": "1",
    "phone": "+79991234567"
}

Все поля обязательны. Телефон должен содержать только цифры и опционально знак + в начале.

#### Корзина

**GET** /cart/ — Просмотр содержимого корзины.

**POST** /cart/ — Добавить товар в корзину.

**PUT** /cart/ — Изменить количество товара.

**DELETE** /cart/ — Удалить товар из корзины.

Требуется авторизация.

**Тело POST / PUT запроса:**

{
    "product_info_id": 66,
    "quantity": 2
}

**Ответ GET (200 OK):**

{
    "cart": [
        {
            "id": 7,
            "product_name": "Товар",
            "shop_name": "Магазин",
            "quantity": 2,
            "price": 120,
            "total": 240
        }
    ],
    "total_sum": 240
}

### 5. Заказы

#### Оформление заказа

**POST** /order/create/

Переводит корзину в статус заказа и создает запись в БД.

Требуется токен.

**Тело запроса:**

{
    "contact_id": 2
}

**Ответ (200 OK):**

{
    "message": "Заказ оформлен",
    "order_id": 8
}

#### Список заказов пользователя

**GET** /orders/

Возвращает историю заказов текущего пользователя (исключая корзину).

Требуется токен.

**Ответ (200 OK):**

[
    {
        "id": 3,
        "dt": "2026-05-31T...",
        "state": "new",
        "contact": "Москва, Тверская 1",
        "total": 240
    }
]

#### Детали заказа

**GET** /orders/{pk}/

Возвращает полную информацию о заказе, включая список товаров.

Требуется токен.

#### Отмена заказа (покупатель)

**POST** /orders/{pk}/cancel/

Требуется токен.

Отменяет заказ. Доступно только владельцу заказа.

**Ответ (200 OK):**

{
    "message": "Заказ отменён"
}

### 6. Функции Поставщика (Supplier)

#### Импорт прайс-листа (Асинхронный)

**POST** /import/

Запускает фоновую задачу импорта товаров из YAML-файла, привязанного к магазину в админке.

Требования: Токен поставщика.

**Тело запроса:**

{
    "shop_id": 2
}

**Ответ (200 OK):**

{
    "message": "Задача импорта для магазина ЗооМир-2 запущена"
}

#### Список новых заказов для поставщика

**GET** /supplier/orders/

Возвращает заказы со статусом new, которые содержат товары из магазинов текущего поставщика.

#### Изменение статуса заказа (Частичное подтверждение)

**PATCH** /orders/{pk}/status/

Поставщик подтверждает наличие своих товаров в заказе. Заказ переходит в статус confirmed только после подтверждения 
всеми участвующими поставщиками.

Требуется токен поставщика.

**Тело запроса:**

{
    "state": "confirmed"
}

(Поле state может игнорироваться, так как логика автоматическая, но эндпоинт используется для триггера действия).

### 7. Функции Кладовщика (Storekeeper)

#### Список заказов для сборки

**GET** /storekeeper/orders/

Возвращает заказы, готовые к работе (статусы confirmed, assembled, sent по умолчанию).

**Параметры фильтрации:**

- status — список статусов (можно передать несколько раз: ?status=confirmed&status=assembled).
- date_from, date_to — фильтр по дате (формат YYYY-MM-DD).
- shop — ID магазина.

#### Изменение статуса заказа (Кладовщик)

**PATCH** /storekeeper/orders/{pk}/status/

Изменяет статус заказа в соответствии с жизненным циклом:
confirmed → assembled → sent → delivered.

**Тело запроса:**

{
    "state": "assembled"
}

#### Экспорт заказов в CSV (Асинхронный)

**GET** /storekeeper/orders/export/

Генерирует CSV-файл с отчетом по заказам.

Параметры фильтрации: Аналогичны списку заказов (status, date_from, date_to, shop).

**Ответ:**

Файл .csv скачивается автоматически. Содержит колонки: ID, Дата, Статус, Клиент, Телефон, Адрес, Сумма, Товары.

#### Экспорт товаров в CSV (Асинхронный)

**GET** /export/products/

Генерирует CSV-файл со списком товаров.

Доступ: Кладовщик, Поставщик (только свои товары), Администратор (все товары).

**Параметры фильтрации:**

- shop — ID магазина.
- category — ID категории.
- min_quantity — минимальный остаток на складе.

**Ответ:**

Файл .csv скачивается автоматически. Содержит колонки: ID, Название, Магазин, Цена, Количество, Параметры.
