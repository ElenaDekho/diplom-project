# API Документация

Базовый URL: `http://127.0.0.1:8000/api/`

## Аутентификация
Токен передаётся в заголовке: `Authorization: Token <token>`

---

## Регистрация
**POST** `/api/register/`

**Тело запроса:**

json
{
    "email": "user@example.com",
    "password": "123456",
    "first_name": "Иван",
    "last_name": "Иванов"
}

**Ответ:**

json
{
    "message": "Пользователь создан"
}

После успешной регистрации на указанный email 
отправляется ссылка для подтверждения.

## Подтверждение регистрации

GET /api/confirm-email/{token}/

**Ответ:**

json
{
    "message": "Email подтверждён"
}

Токен действителен 24 часа. 
После подтверждения пользователь становится активным.

## Импорт товаров через админку

## Кладовщик (storekeeper)

### Список заказов для сборки
**GET** `/api/storekeeper/orders/`

**Параметры фильтрации:**
- `status` — список статусов (по умолчанию: `confirmed`, `assembled`, `sent`)
- `date_from`, `date_to` — фильтр по дате (YYYY-MM-DD)
- `shop` — ID магазина

**Ответ:** список заказов с полями: `id`, `dt`, `state`, `contact`, `total`.

---

### Изменение статуса заказа
**PATCH** `/api/storekeeper/orders/<id>/status/`

**Тело запроса:**

json
{
    "state": "assembled"   // или "sent", "delivered"
}

Последовательность статусов:
confirmed → assembled → sent → delivered (обратный переход запрещён)

## Асинхронный экспорт заказов (кладовщик)

**GET** `/api/storekeeper/orders/export/`

**Параметры фильтрации:**

- status (по умолчанию: confirmed, assembled, sent)

Пример: `?status=confirmed&status=assembled`

- date_from, date_to

- shop — ID магазина

**Примечание:** Если параметр `status` не указан, выгружаются заказы только со статусами `confirmed`, `assembled`, `sent` 
(готовые к сборке, собранные и отправленные). Статус `new` (новый) не выгружается по умолчанию, 
но может быть явно запрошен.

**Ответ:** 
CSV-файл со списком заказов с колонками: ID заказа, Дата, Статус, Клиент, Телефон, Адрес, Сумма, Товары 
(генерируется асинхронно, возвращается сразу после готовности).

## Асинхронный экспорт товаров

**GET** `/api/export/products/`

**Доступ:** кладовщик, поставщик (только свои товары), администратор.

**Параметры фильтрации:**

- shop — ID магазина
- category — ID категории
- min_quantity — минимальное количество на складе

**Ответ:**

CSV-файл со списком товаров с колонками: ID, Название товара, Магазин, Цена, Количество, Параметры 
(генерируется асинхронно, возвращается сразу после готовности)

## Частичное подтверждение заказа поставщиками

При заказе товаров от нескольких поставщиков каждый поставщик подтверждает свою часть через PATCH /api/orders/<id>/status/. Заказ переходит в статус confirmed только после подтверждения всех участвующих поставщиков.

## Валидация контактов

При создании контакта все поля (city, street, house, phone) обязательны. При пустых полях возвращаются сообщения:

- Укажите город
- Укажите улицу
- Укажите номер дома
- Укажите телефон

Телефон должен содержать только цифры и может начинаться с +.
