# Сервис автоматизации закупок

**Дипломная работа по специальности "Python-разработчик"**

**Автор:** Елена Дэхо

## Описание проекта

Сервис представляет собой платформу для автоматизации процессов закупки и продажи товаров. 
Система связывает покупателей, поставщиков и кладовщиков в едином информационном пространстве, 
обеспечивая асинхронную обработку заказов, управление остатками и уведомление участников через email.

### Роли и функционал

#### 🛒 Покупатель

- **Каталог:** Просмотр товаров с фильтрацией по магазинам, категориям и поиском по названию.
- **Избранное:** Сохранение понравившихся товаров для последующей покупки. Возможность перемещения товаров из избранного в корзину.
- **Корзина:** Добавление товаров из разных магазинов. Изменение количества позиций. Валидация наличия товара и статуса магазина (принимает ли заказы).
- **Контакты:** Управление адресами доставки (до 5 контактов на пользователя).
- **Заказы:** Оформление заказа, просмотр истории, отмена активных заказов. Получение уведомлений о смене статусов.

#### 🏭 Поставщик

- **Управление магазинами:** Создание нескольких магазинов. Включение/выключение приема заказов (state).
- **Импорт товаров:** Асинхронная загрузка прайс-листов из YAML-файлов. Поддержка динамических характеристик товаров.
- **Обработка заказов:** Просмотр новых заказов, содержащих товары его магазинов. Частичное подтверждение заказа (подтверждает только свою часть). Заказ переходит в статус «Подтвержден» только после подтверждения всеми участвующими поставщиками.
- **Экспорт:** Выгрузка списка своих товаров в CSV.

#### 📦 Кладовщик

- **Управление сборкой:** Просмотр заказов со статусами confirmed, assembled, sent.
- **Смена статусов:** Последовательное изменение статуса заказа: confirmed → assembled → sent → delivered.
- **Экспорт:** Выгрузка отчетов по заказам в CSV с фильтрацией по датам, статусам и магазинам.

#### 🔑 Администратор
- Полный доступ к управлению данными через Django Admin.
- Возможность запуска массового импорта товаров для выбранных магазинов через админ-панель.
- Экспорт всех товаров системы.

### Технический стек

- **Backend:** Python 3.11, Django 5.2, Django REST Framework
- **База данных:** SQLite (для разработки), поддержка PostgreSQL
- **Асинхронные задачи:** Celery 5.6.3 + Redis (брокер сообщений)
- **Документация API:** drf-spectacular (OpenAPI 3.0 / Swagger UI)
- **Контейнеризация:** Docker, Docker Compose
- **Тестирование:** Pytest, Coverage (94%)

### Архитектура и ключевые решения

#### 1. Асинхронность (Celery + Redis)

Тяжелые операции вынесены в фоновые задачи, чтобы избежать блокирования основного потока приложения и обеспечить быстрый отклик интерфейса:

- **Импорт товаров:** Парсинг YAML и обновление БД.
- **Экспорт данных:** Генерация CSV-файлов больших объемов.
- **Отправка Email:** Уведомления о регистрации, заказе и смене статусов.

#### 2. Динамические характеристики товаров

Реализована гибкая система атрибутов товаров через модели Parameter и ProductParameter.

- Характеристики не жестко заданы в коде, а добавляются динамически при импорте из YAML.
- Это позволяет хранить разные параметры для разных категорий (например, "цвет" для одежды и "диагональ" для телевизоров) в одной структуре.

#### 3. Заказы с товарами от разных поставщиков

Один заказ может содержать товары из разных магазинов, от разных поставщиков.

- При оформлении заказа создается запись со статусом new.
- Каждый поставщик подтверждает наличие своих товаров через API.
- Заказ переходит в статус confirmed только после подтверждения всеми участвующими поставщиками.
- После этого заказ становится доступен кладовщику для сборки.

#### 4. Управление остатками

- При оформлении заказа количество товара (quantity) в модели ProductInfo уменьшается на величину заказа.
- Поставщик может актуализировать остатки в любой момент через повторный импорт YAML-файла.

#### 5. Импорт через Админку

Помимо API-эндпоинта для поставщиков, реализовано действие в Django Admin для модели Shop:

«Запустить асинхронный импорт для выбранных магазинов»

Администратор может выбрать несколько магазинов в списке и запустить импорт их прайс-листов одной кнопкой. Задача выполняется асинхронно через Celery.

#### 6. Отправка Email

В текущей конфигурации используется консольный бэкенд (django.core.mail.backends.console.EmailBackend) для вывода писем в stdout при разработке.
Для перехода на продакшен достаточно изменить настройки EMAIL_BACKEND в settings.py на SMTP-сервер (Yandex, Gmail и др.), указав хост, порт и учетные данные в переменных окружения.

#### 7. Логирование

Настроено логирование ошибок приложения и задач Celery:

- Вывод критических ошибок в консоль.
- Запись подробных логов в файл celery_errors.log для последующего анализа.

## Установка и запуск

### Требования

- Docker & Docker Compose
- Git

### Быстрый старт через Docker

1. Клонируйте репозиторий:

git clone https://github.com/ElenaDekho/diplom-project.git

cd diplom-project

2. Создайте файл .env в корневой папке (пример содержимого):

SECRET_KEY=your-secret-key-here

CELERY_BROKER_URL=redis://redis:6379/0

CELERY_RESULT_BACKEND=redis://redis:6379/0

EMAIL_HOST_USER=your-email@example.com

EMAIL_HOST_PASSWORD=your-password

3. Запустите контейнеры:

docker compose up -d --build

4. Примените миграции и создайте суперпользователя:

docker compose exec web python manage.py migrate

docker compose exec web python manage.py createsuperuser

5. Сервис доступен по адресу: http://localhost:8000

- **Admin Panel:** http://localhost:8000/admin/
  
- **Swagger API Docs:** http://localhost:8000/api/docs/

**Важно:** Если вы изменяли модели (models.py) после первого запуска, не забудьте создать новые миграции перед их применением:

docker compose exec web python manage.py makemigrations

docker compose exec web python manage.py migrate

### Локальный запуск (без Docker)

1. Установите зависимости:

pip install -r requirements.txt

2. Примените миграции:

python manage.py migrate

3. Запустите Redis (требуется установленный Redis Server):

redis-server

4. Запустите сервер Django:

python manage.py runserver

5. В отдельном терминале запустите Celery Worker:

celery -A config worker --loglevel=info
